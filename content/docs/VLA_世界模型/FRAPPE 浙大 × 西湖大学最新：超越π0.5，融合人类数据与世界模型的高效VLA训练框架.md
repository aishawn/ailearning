# FRAPPE：世界建模注入通用策略的多未来表征对齐

浙大、西湖大学、港科大（广州）、华南理工、上科大、清华等提出 **FRAPPE**（Future Representation Alignment via Parallel Progressive Expansion），将世界建模能力注入 VLA 策略，在 RoboTwin 与真机任务上超越 RDT-1B、π0.5，并在长时程与未见场景中表现更强泛化。

- **论文**：https://arxiv.org/abs/2602.17259  
- **主页**：https://h-zhao1997.github.io/frappe/

---

## 问题与思路

- **问题**：现有方法（1）训练目标过度强调像素级重建，限制语义与泛化；（2）推理依赖预测的未来观测，易误差累积。
- **方案**：通过**并行渐进扩展**实现多未来表征对齐——不做显式像素级未来生成，而是对**未来观测的隐空间表征**做对齐，使动作与视觉在统一语义空间中理解；同时用**并行扩展**（多输入流、多教师表征）避免单一归纳偏置。

---

## 方法要点

- **训练机制**：隐式世界建模，对齐未来观测的隐空间表征。
- **模型结构**：将单流 VLA 扩展为多流并行前向，每流对齐一独立视觉编码器（实验采用 3 路：DINOv2、CLIP、ViT），多教师表征联合学习。
- **训练阶段**：Mid-Training 全参数微调、与 Theia 对齐；Post-Training 引入并行扩展（多 prefix + LoRA），转为混合专家并做多教师特征对齐。

---

## 实验要点

- **仿真**：RoboTwin 8 子任务平均成功率超越 naive fine-tuning 的 RDT-1B、π0、π0.5 及 DP、VPP；RDT-130M 经 FRAPPE 后与 RDT-1B 持平，小模型也受益。
- **真机**：四类双臂操作任务上超过 RDT-1B 与 π0.5，未见复杂场景泛化更好；长程三子任务中 FRAPPE 与 π0.5 相当，RDT-1B 无法按序完成。
- **人类数据**：Ego (Web) 与 Ego (Task) 人类视频均可提升下游成功率，降低遥操作数据需求。
- **效率**：并行扩展带来额外参数与计算，推理延迟仍与 RDT-1B 接近，显存可接受。

FRAPPE 通过表征对齐与多源数据融合，在可接受推理开销下提升泛化、长时序与隐式世界建模能力。

## 方法详解：并行扩展与两阶段训练

FRAPPE 的架构设计非常精妙，它在保持基础模型（如 RDT-1B）冻结的情况下，通过引入轻量级的模块实现了能力的跃迁。

### 核心架构：MiPA 与并行对齐

作者提出了一种名为 **MiPA**（**混合前缀与 LoRA，Mixture-of-Prefix-and-LoRA**）的架构。具体来说，在 RDT 的**扩散 Transformer（Diffusion Transformer, DiT）**主干网络中，除了输入当前的观测、指令和噪声动作，还额外加入了一组可学习的**未来前缀（Future Prefix）**令牌 。

![FRAPPE 的训练与推理流程概览。在训练阶段，模型通过两阶段过程学习与多个视觉基础模型的表征空间对齐；在推理阶段，通过 Router 聚合多个专家流的输出。](https://pub-adba99cbc4cd4237a5ed7de21ad26f3c.r2.dev/md-img/92682798f0050411.png)

FRAPPE 的训练与推理流程概览。在训练阶段，模型通过两阶段过程学习与多个视觉基础模型的表征空间对齐；在推理阶段，通过 Router 聚合多个专家流的输出。

- **输入（Input）**：语言指令 、当前观测 （由 SigLIP 编码）、噪声动作  以及新引入的未来前缀 。
    
- **输出（Output）**：预测的动作块  和对齐后的未来表征 。
    

为了避免单一视觉模型的归纳偏差，FRAPPE 同时对齐了三个强大的视觉老师：CLIP、DINOv2 和 ViT。每个老师对应一个“专家流”，包含独立的 Prefix 和 LoRA 模块。最后，通过一个轻量级的**路由网络（Router）**来聚合这些专家的输出。

### 负载均衡与两阶段渐进式训练

在并行训练多个专家时，很容易出现“一家独大”的坍缩现象。为此，FRAPPE 引入了**负载均衡损失（Load-balancing Loss）**，通过惩罚路由权重的方差来鼓励各个专家流均衡更新。同时，还采用了**标签平滑（Label Smoothing）**技术，确保即使在训练初期，每个专家也能获得非零的梯度。

为了让模型平稳过渡到世界模型任务，作者设计了两个阶段：

1. **Mid-training（中段训练）**：先用一个从多个 VFM 中蒸馏出来的“小老师”（如 86M 的 Theia）进行全参数微调，让模型初步建立起预测未来的意识。
    
2. **Post-training（后段训练）**：引入并行扩展，利用 LoRA 和 Prefix 对齐多个具体的 VFM 老师。

## 实验结果：刷新 SOTA，长程任务显神威

研究团队在 RoboTwin 仿真基准和真实世界中对 FRAPPE 进行了全方位的测试。

### 仿真表现：Hard 模式下的绝对优势

在 RoboTwin 2.0 基准测试中，FRAPPE 与 DP、VPP、RDT 以及最新的 、 进行了对比。

![在 RoboTwin 2.0 基准上的对比。FRAPPE 在 Easy 任务上表现稳健，在 Hard 任务（包含随机背景、光照和高度变化）中显著优于其他模型。](https://pub-adba99cbc4cd4237a5ed7de21ad26f3c.r2.dev/md-img/2de53c685f91bf19.png)

在 RoboTwin 2.0 基准上的对比。FRAPPE 在 Easy 任务上表现稳健，在 Hard 任务（包含随机背景、光照和高度变化）中显著优于其他模型。

实验数据显示，在最具挑战性的 **Hard 设置**下，FRAPPE 的平均成功率达到了 **25.5%**，远超 RDT 的 15.1% 和  的 13.3%。此外，这种架构在 130M 的小模型上也表现出了极强的生命力。

![不同参数规模下的表现。即便是 130M 的小模型，在使用了 FRAPPE 微调后，也能在某些任务上叫板 1B 规模的基准模型。](https://pub-adba99cbc4cd4237a5ed7de21ad26f3c.r2.dev/md-img/67c80aa9daed69e1.png)

### 推理效率：性能倍增，延迟微增

很多人担心并行扩展会拖慢推理速度。但 FRAPPE 证明了这种担忧是不必要的。

![推理效率对比。可以看到，即使增加了并行专家，在 5 步去噪下，延迟也仅从 0.214s 增加到 0.235s，显存占用 8.0GB，单张 3090 或 4090 即可轻松运行。](https://pub-adba99cbc4cd4237a5ed7de21ad26f3c.r2.dev/md-img/30f23463ff0771c8.png)

推理效率对比。可以看到，即使增加了并行专家，在 5 步去噪下，延迟也仅从 0.214s 增加到 0.235s，显存占用 8.0GB，单张 3090 或 4090 即可轻松运行。

更有意思的是，当把去噪步数减少到 3 步时，FRAPPE 的性能依然优于 5 步的 RDT，而延迟却降低到了 **0.173s**。

### 真实世界：从 0% 到 20% 的突破

在真实世界的长程任务（如：抓取玉米 -> 倒水 -> 盖盖子）中，基础模型 RDT 因为无法处理精细操作和动作连续性，成功率为 0%。而 FRAPPE 凭借强大的环境感知能力，实现了 **20%** 的完整任务成功率。

## 协同训练：“零成本利用”人类视频数据

FRAPPE 还有一个非常实用的特性：它可以利用**无动作标签的人类视频**进行训练。作者构建了一个“数据金字塔”：

- **底层**：大规模互联网人类视频（如 TASTE-Rob 数据集）。
    
- **中层**：任务相关的人类操作视频（无需机器人遥操作，采集效率极高）。
    
- **顶层**：少量的机器人专家数据。
    

![FRAPPE 的数据金字塔模型。通过分层利用数据，模型可以从人类视频中汲取丰富的物理先验。](https://pub-adba99cbc4cd4237a5ed7de21ad26f3c.r2.dev/md-img/0c65d60d09fa9ca7.png)

FRAPPE 的数据金字塔模型。通过分层利用数据，模型可以从人类视频中汲取丰富的物理先验。

![利用人类自我中心视频数据。通过在训练中加入人类操作视频，模型在处理从未见过的物体和复杂几何形状时表现出更强的归纳先验。](https://pub-adba99cbc4cd4237a5ed7de21ad26f3c.r2.dev/md-img/aedd47607d591681.png)

利用人类自我中心视频数据。通过在训练中加入人类操作视频，模型在处理从未见过的物体和复杂几何形状时表现出更强的归纳先验。

实验证明，即使机器人轨迹非常稀少（仅 5 条），只要加入人类视频进行未来表征对齐训练，模型在处理复杂物体（如大蒜、方块）时的成功率就会大幅提升。
