# 文章编辑管理 - 功能说明与检测

本文档说明本站**所有与文章/文档编辑、管理相关**的入口、流程和注意事项。

---

## 一、内容类型总览

| 类型 | 前台访问地址 | 数据来源 | 后台管理入口 |
|------|--------------|----------|--------------|
| **文档 (Docs)** | `/docs/...` | ① 数据库 ② 文件 `content/docs/*.md` | Admin → Docs / 内容文件(MD) |
| **文章 (Posts)** | `/blog/...` | 数据库 | Admin → Posts |

**文档**有两种来源，访问 `/docs/xxx` 时的规则是：**先查数据库，没有再读文件**。

---

## 二、文档（Docs）的两种管理方式

### 方式 A：数据库文档（Admin → Docs）

- **入口**：登录后 **Admin → 页面与内容 → Docs**（或 `/admin/docs`）。
- **功能**：
  - **列表**：查看所有「数据库里的文档」，支持分页。
  - **新增**：`/admin/docs/add`，填写 slug、标题、描述、正文（Markdown）。
  - **编辑**：列表每行「编辑」→ `/admin/docs/[id]/edit`。
  - **查看**：列表每行「查看」→ 新标签打开 `/docs/[slug]`。
- **编辑器**：Markdown 编辑器，支持 **粘贴 / 拖拽图片**，自动上传到站内存储并插入 `![](url)`。
- **存储**：数据库表（Post，type=DOC）。**不会**写入 `content/docs` 目录。
- **前台**：若某 slug 在数据库存在，则 `/docs/该slug` 只显示数据库内容；**不会**再读同名文件。
- **侧栏**：本站 Docs 侧栏来自 **文件结构**（fumadocs）。因此「仅存在数据库、没有对应文件」的文档，**不会出现在侧栏**，只能通过直接链接或搜索访问。

**适合**：单篇独立文档、活动页、不依赖目录结构的说明。

---

### 方式 B：文件文档（Admin → 内容文件 (MD)）

- **入口**：**Admin → 页面与内容 → 内容文件 (MD)**（或 `/admin/content-editor`）。
- **功能**：
  - **下拉选择**：列出 `content/docs` 下所有 `.md` / `.mdx` 文件（含子目录）。
  - **编辑**：选文件后加载内容，在 Markdown 编辑器中修改；支持 **粘贴 / 拖拽 / 点击「插入图片」**，图片上传后自动插入。
  - **保存**：点击「保存」将内容**写回原文件**（覆盖 `content/docs` 下对应文件）。
  - **在站点中查看**：当前选中的文件会有一个「在站点中查看」链接，新开标签打开该文档在前台的 `/docs/...` 地址。
- **存储**：直接修改仓库里的 `content/docs/**/*.md`、`*.mdx`。
- **前台**：当某个路径**没有**在数据库里命中时，会从 `content/docs` 按路径读文件并渲染（fumadocs）。
- **侧栏**：Docs 左侧目录树由 **content/docs 的文件结构** 生成，所以这里编辑的文件会出现在侧栏。

**适合**：你现有的「论文解读」等按目录组织的文档（如 `VLA_世界模型/xxx.md`）。

---

## 三、文章（Posts / 博客）

- **入口**：**Admin → 页面与内容 → Posts**（或 `/admin/posts`）。
- **功能**：列表、新增、编辑（含 Markdown 正文）；同样支持 **粘贴 / 拖拽图片** 自动上传并插入。
- **前台**：通过 `/blog/...` 访问，与 `/docs/...` 分离。

---

## 四、如何选择编辑入口（速查）

| 你要做的事 | 使用入口 |
|------------|----------|
| 编辑已有论文解读（如 `content/docs/VLA_世界模型/xxx.md`） | **Admin → 内容文件 (MD)**，下拉选该文件后编辑、保存。 |
| 新增一篇「不跟文件目录绑定的」文档 | **Admin → Docs → 添加**，填 slug/标题/正文，图片可粘贴。 |
| 编辑已在「Admin → Docs」里建过的某篇文档 | **Admin → Docs**，列表中找到该文档点「编辑」。 |
| 写博客文章 | **Admin → Posts**，新增或编辑。 |

---

## 五、图片处理（所有编辑入口统一）

- **Admin → Docs**（数据库文档）、**Admin → Posts**、**Admin → 内容文件 (MD)** 中的 Markdown 编辑器均支持：
  - **粘贴图片**（Ctrl+V）
  - **拖拽图片**到编辑器
  - **内容文件 (MD)** 额外提供「插入图片」按钮
- 图片会先上传到站内配置的存储（如 OSS/R2），再在正文中插入 `![](url)`，无需手写路径。

---

## 六、权限

- **Admin → Docs**（含添加/编辑）：需要 **文档写权限**（`admin.docs.write`）。
- **Admin → 内容文件 (MD)**：同样需要 **文档写权限**。
- **Admin → Posts**：需要 **文章写权限**（`admin.posts.write`）。

---

## 七、已知限制与注意点

1. **数据库文档不在侧栏**：只在 Admin 里新建、未在 `content/docs` 建对应文件的文档，不会出现在 Docs 左侧目录树，需通过链接或搜索访问。
2. **同名 slug 优先数据库**：若数据库存在 slug `vla/xxx`，则即使有 `content/docs/vla/xxx.md`，前台 `/docs/vla/xxx` 也只显示数据库内容。
3. **内容文件 (MD) 保存即写盘**：保存会直接覆盖 `content/docs` 下文件，建议配合 Git 使用，便于回滚。
4. **「在站点中查看」**：使用当前选中文件的路径生成 `/docs/...` 链接；若站点启用了 locale 前缀，新开标签后可能需手动切语言，取决于你的 i18n 配置。

---

## 八、功能检测清单（自检用）

- [ ] **Admin → Docs**：能打开列表、新增、编辑、查看；编辑时粘贴图片能上传并插入。
- [ ] **Admin → 内容文件 (MD)**：下拉能列出 `content/docs` 下的 .md/.mdx；选择后能编辑、保存；粘贴/拖拽/插入图片正常；「在站点中查看」能打开对应文档页。
- [ ] **Admin → Posts**：能新增、编辑；粘贴/拖拽图片能上传并插入。
- [ ] **前台**：`/docs/xxx` 对「仅文件」「仅数据库」「两者都有」的 slug 行为符合上文说明；侧栏只显示文件结构下的文档。

按上述入口和清单操作，即可完整覆盖本站的文章编辑与管理功能。
