# VLA 不精准原因分析

## 一、VLA 不精准的核心原因

我从 6 个层面讲：感知 → 表征 → 推理 → 动作 → 训练 → 系统工程。

1️⃣ 视觉感知是 2D 语义驱动，不是真 3D 几何

当前主流 VLA（如基于 Qwen2.5-VL、LLaVA 的模型）本质是：

强语义理解模型 + 弱几何理解

问题：

深度误差

尺寸估计不准

物体 6D pose 不稳定

遮挡场景下失效

它们理解“这是什么”，但不理解“它在三维空间中具体在哪里”。

结果：

抓取偏 1–2 cm 就会失败。

2️⃣ 视觉 token 是离散语义，不是物理连续空间

VLM 的视觉 token：

是 patch embedding

不是 metric 空间表示

没有真实尺度

机器人控制需要的是：

连续空间

可微物理

精确坐标

两者之间存在 representation gap。

3️⃣ 动作学习是模仿驱动，不是物理建模

很多 VLA 训练方式：

行为克隆（BC）

imitation learning

next-token action prediction

缺点：

误差累积

长时任务漂移

无物理约束

而不是：

模型预测控制（MPC）

可微物理

接触建模

4️⃣ 数据问题：机器人数据太少

对比：

语言模型：万亿 token

视觉模型：亿级图像

机器人操作数据：百万级以下

数据分布极度稀疏：

长尾物体

真实接触

摩擦变化

柔性物体

导致泛化差。

5️⃣ 误差没有闭环反馈

LLM/VLM 是 open-loop。

机器人世界需要：

状态反馈

视觉更新

接触反馈

实时校正

但很多 VLA 是：

图像 → 语言理解 → 一步生成动作

而不是：

感知 → 预测 → 执行 → 再感知 → 修正
6️⃣ 多模态对齐不精确

语言空间 ≠ 视觉空间 ≠ 动作空间

目前多模态对齐是：

contrastive learning

projection layer

cross attention

但没有保证：

几何一致

物理一致

时序一致

二、典型表现问题
现象	本质原因
抓偏	深度不准
放置歪	6D pose 不准
推动物体滑动异常	无摩擦建模
多步任务崩溃	误差累积
遮挡失败	无 3D 结构理解
长时操作漂移	无闭环控制
三、解决方向（当前研究趋势）
1️⃣ 引入 3D 几何基础模型

如：

VGGT

点云 transformer

NeRF-based 表征

目标：

把 2D VLM 升级为 3D-aware VLA

方法：

几何蒸馏

深度对齐

3D token 融合

多视图输入

这是当前非常重要的方向。

2️⃣ 从“语义对齐”升级为“几何对齐”

不是：

图像 ↔ 语言

而是：

2D视觉 ↔ 3D几何 ↔ 语言 ↔ 动作

研究方向包括：

3D latent alignment

结构一致性损失

epipolar constraint regularization

3️⃣ 引入闭环控制（Closed-loop VLA）

升级为：

感知-预测-执行-修正循环

加入 state estimator

使用视觉伺服（visual servoing）

即：

预测动作
↓
执行
↓
观察误差
↓
微调动作
4️⃣ 结合模型预测控制（MPC）

把 LLM 用作：

高层规划器

把低层控制交给：

MPC

PID

可微物理引擎

这是一种层级架构：

LLM 规划
↓
VLA 生成中层策略
↓
MPC 精细执行
5️⃣ 大规模真实机器人数据

方向包括：

多机器人数据共享

模拟器 + 真实联合训练

domain randomization

自监督交互数据

比如：

Google RT 系列

Open X-Embodiment

（如果你需要我可以详细对比）

6️⃣ 从 Token 预测转向世界模型

未来趋势：

不直接预测动作，而是预测世界状态变化。

即：

当前状态 → 预测未来状态 → 选择动作

这更接近物理推理。

四、未来趋势总结
阶段	特点
VLA 1.0	语义驱动
VLA 2.0	3D 几何增强
VLA 3.0	闭环控制
VLA 4.0	世界模型驱动

精准度的提升核心在于：

让模型理解“空间”和“物理”，而不仅仅是“语义”。

五、关键一句话总结

VLA 不精准的根本原因是：

语义智能强，空间物理智能弱。

解决之道是：

几何建模 + 闭环控制 + 物理约束 + 世界模型。
