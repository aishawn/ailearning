# 论文 / 文档编辑与管理说明

> **完整功能说明**（含文档 vs 文章、数据库 vs 文件、权限与自检清单）见：[文章编辑管理-功能说明](./文章编辑管理-功能说明.md)。

本站内容存放在 `content/docs` 下的 Markdown（.md/.mdx）文件中。下面两种方式都可以用来舒服地编辑和管理（包括图片）。

## 方式一：站内编辑器（推荐）

1. 登录后进入 **Admin → 页面与内容 → 内容文件 (MD)**（或直接访问 `/admin/content-editor`）。
2. 在「选择文档」下拉框中选择要编辑的文件（如 `VLA_世界模型/字节&港大提出WoG….md`）。
3. 在编辑器中修改内容：
   - **图片**：可直接粘贴（Ctrl+V）、拖拽图片到编辑器，或点击「插入图片」选择文件；图片会先上传到站内存储，并自动插入 `![](url)`。
   - 支持实时预览、保存到原文件。
4. 修改后点击「保存」，内容会写回对应的 `.md` 文件。

**说明**：仅限有「文档写权限」的管理员使用；保存会直接改写仓库里的文件。

## 方式二：本地撰写后部署（推荐给习惯本地写的人）

在本地用编辑器写/改 MD，图片放好，再 `git push` 重新部署即可。**目录约定、图片路径、编辑器选择**见：

👉 **[本地撰写与部署指南](./本地撰写与部署指南.md)**

简要约定：

- **正文**：只改 `content/docs/` 下的 `.md` / `.mdx`。
- **图片**：统一放 `public/docs-images/`（可按主题建子目录，如 `VLA_世界模型/`），文中用绝对路径：`![说明](/docs-images/主题/文件名.png)`。
- **编辑器**：Typora（粘贴即存图）、Obsidian（知识库+双链）、VS Code（Paste Image 等插件）任选，把「粘贴/插入图片」的目标目录设到 `public/docs-images` 即可。

## 小结

- **想少折腾、在浏览器里搞定**：用 **Admin → 内容文件 (MD)**，支持粘贴/拖拽/插入图片并保存到文件。
- **想用 Notion/OneNote 那种块编辑**：目前站内是「MD 源码 + 实时预览」；若以后要块编辑，可再接入 Tiptap/BlockNote 等。
- **想用本地软件**：用 Typora / Obsidian / VS Code 直接编辑 `content/docs`，图片按上面约定放好即可。
