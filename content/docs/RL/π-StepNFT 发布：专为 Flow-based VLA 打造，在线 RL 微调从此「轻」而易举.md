## 背景：Flow-based VLA 很强，但 Online RL 被“似然与泛化”卡住

近几年，VLA 已从离散 Token 演进至连续 Flow-matching 策略，建立了强操作先验。但系统性分析表明，**SFT 收敛后策略输出易塌缩至单一模式**，模型仅沿狭窄的“专家流形”工作，缺乏执行偏移时的局部纠错能力。因此，Online RL 的目标并非从零学控制，而是**扩展专家轨迹附近的“可恢复流形”**，赋予策略抗偏移能力。

然而，Flow-based VLA 的 Online RL 面临核心瓶颈：策略通过多步 ODE 求解生成动作，导致**动作似然与梯度计算昂贵且不稳定**。为绕开似然问题，现有路线大致分两类：一是**绕开似然**，通过价值蒸馏或偏好反馈避免计算对数似然（如 GR-RL，π*0.6），但需要提前学习一个价值网络；二是**近似似然**，把 ODE 转成 SDE 或引入高斯参数化（如 πRL），但需要多次前向以构建起联合概率通路。

## 动机：Diffusion-NFT 为何在图像生成很强，迁移到具身控制却遭遇瓶颈？

π-StepNFT 受到图像生成领域 **Diffusion-NFT** 的启发：它提供了一条“无似然、高效率”的诱人路径。但直接迁移到具身控制时效果不佳（成功率仅到0.7+），核心在于两类任务的 **“物理属性”存在本质差异：**

- **图像生成：静态分布匹配**
    

关心的是把噪声映射到最终去噪样本 ，本质是匹配静态分布。在确定性 **ODE 采样** 下，去噪中间状态沿稳定轨迹前进，使用“终点  回归”通常是良定的。

- **具身控制：序列决策 + 闭环执行**
    

不是一次性生成“作品”，而是在环境里滚动执行。**任何一步动作的偏差都会改变下一步观测**，误差会逐步累积（复利效应）。在这种序列闭环下，若仍像图像生成那样只对最终做粗粒度监督，策略在 on-policy 漂移时很难被及时“拉回”。

### 📌  Domain gap 的“两大关键因素”

本文把迁移失败归结为两个核心矛盾：

- **缺乏探索 (Lack of Exploration)：**确定性 **ODE rollout** 很快塌缩到狭窄流形，难在高维动作空间里找到多样解。
    
- **监督目标不匹配 (Supervision Target Mismatch)：**一旦用 **SDE 注入噪声** 扩大探索，噪声会沿去噪路径累积。若仍用终点  做监督，会导致高方差、点对点不稳定对齐。
    

### 📌 一个关键前提：具身控制通常采用更短的去噪路径

更重要的是，具身控制有实时交互延迟约束，通常只用 **很短的去噪路径**；并且已有实证结果指出：把路径“适度变长”收益并不会增加。这使得我们能够在工程上可行地做 **细粒度的 step-wise 监督**。这条“更短去噪路径”的事实，是后续实现 **单次前向 + 逐步监督** 的关键前提。

## 方法：两阶段流程 × 三项核心机制

π-StepNFT 的整体算法流程分为 **轨迹采样** 与 **策略更新** 两个阶段，通过三项核心机制实现高效、稳定的无似然强化学习：

### 📌 整体流程：采样与更新闭环

- **数据采样阶段**：使用旧策略  在环境中 rollout。对每个环境步，运行 Flow-SDE 采样得到一条去噪链，**随机抽取一个去噪转移对**并记录旧策略输出 ，轨迹结束后获得环境返回的二值回报存入 buffer。
    
- **策略更新阶段**：从 buffer 采样 batch，对每条样本仅做 **一次前向** 得到当前输出，与旧输出做差得到更新方向。沿该方向构造 **正/负镜像分支**，计算它们对真实转移  的解释误差。最后把轨迹回报转成排序标签，用 logistic 排序损失让成功样本倾向于正分支（失败相反），加入 trust-region 正则约束更新幅度，完成梯度更新后 EMA 更新旧策略。
    

### 📌 核心机制 1：Wider Space（Flow-SDE 探索）

ODE 的确定性轨迹把探索空间锁死在初始噪声分布上，策略很快塌到窄流形。π-StepNFT 训练时把采样从 ODE 切换到 **SDE**：在每个去噪步注入受控噪声，让策略能覆盖专家轨迹附近的邻域，形成“更宽的行为流形”。Flow-SDE 的一步更新诱导出一个高斯转移密度，其转移均值是网络输出的仿射函数，从而可以把梯度直接从一步转移目标传回网络参数，避免对 ODE 求解器反传，这是“高效训练”的基础。

### 📌 核心机制 2：Finer Steps（步级监督对齐）

SDE 虽然让探索变宽，但噪声会沿路径累积；此时继续用终点 x0 做监督，相当于要求模型把“抖了一路”的样本一次性拉回某个点，梯度方差很大。π-StepNFT 将监督目标改为 **一步转移**：监督当前去噪状态  走到下一步 （而不是去噪终点 ），并配合方差归一化，得到更稳定、低方差、局部的学习信号。因为具身控制往往只有少量去噪步，我们甚至可以在每个环境步里均匀采样一个去噪转移来做训练样本，进一步降低开销。

### 📌 核心机制 3：Push‒Pull Dynamics（对比排序更新）

沿用 Diffusion-NFT 的 wMSE 目标在具身控制中存在一个“隐式惩罚”，会抑制分支分离幅度，特别是在二值回报时只能进行单边更新。π-StepNFT 改用 **logistic 对比排序损失**，建立清晰的 push‒pull dynamics：对成功轨迹，既“拉近”正分支，也“推远”负分支；对失败轨迹，反向处理。从而在每个 step 上形成更强的判别信号，更新更积极、收敛更稳，且 **无需训练额外的 Critic 网络。**

## 实验验证：Few-shot 解锁潜力 + OOD 泛化更强

论文在 **LIBERO** 与 **ManiSkill** 上做了系统实验，验证了方法的有效性与泛化能力：：

- **LIBERO（少样本潜力解锁）：**在 LIBERO few-shot SFT 设置下显示，不充分的数据训练下的 SFT 模型初始成功率并不高，而 π-StepNFT 通过更宽探索显著提升平均成功率，**对 pi0 的平均提升达 32.9%。**
    

![图片](https://pub-adba99cbc4cd4237a5ed7de21ad26f3c.r2.dev/md-img/306726893eb2513d.png)

- **ManiSkill（OOD 泛化更强）：**ManiSkill 强调视觉多样性与 OOD 测试。论文指出：value-based 方法的 Critic 可能对视觉/语言嵌入发生“多模态过拟合”，导致 OOD 掉点；π-StepNFT 直接依赖环境的成功/失败信号，避免 Critic 过拟合问题，在 OOD 场景表现更强，**在pi0上超越基于 Critic 的基线 11.1%。**

## 组件与消融：为什么它能稳、能快、能泛化？

论文围绕四个关键问题做了系统分析，验证了各组件的必要性：

- **是否真的需要 SDE 的随机探索？**（对比 ODE、SDE 及其修正项，证实 SDE 对流形扩展至关重要）
    
- **监督粒度为什么必须 step-wise？**（终点  vs 逐步 ，证实逐步监督能显著降低方差、稳定对齐）
    
- **排序损失比 wMSE 好在哪？**（移除“隐式分离惩罚”，形成 push–pull，更新更积极）
    
- **必须要 Critic 吗？**（稀疏二值回报也能训练得很稳，避免价值网络过拟合）

## 总结与展望

RL 后训练已被证明是 LLM 能力跃迁的必要路径，但 Flow-based 模型因似然难算，天然适配 RL 困难。而在具身决策领域，RL 又是提升任务泛化性的关键手段。当前主流 VLA 多采用 Flow 架构，这为在环境中用 RL 训练 VLA 模型带来了显著挑战。本文通过深入分析从图像生成到具身控制的 Domain Gap，提出 π-StepNFT，为 Flow-based VLA 的在线强化学习提供了一条高效、稳健、可扩展的新路径。

- **降低部署门槛**：单步前向更新无需反向传播通过求解器，且移除 Critic 网络，显著降低显存与计算开销，使大规模 VLA 微调更易落地。
    
- **为真实部署铺路**：避免 Critic 对多模态特征过拟合，在 OOD 场景下表现更强，提升真实部署的鲁棒性与泛化性。
    
- **开源共享**：基于RLinf代码构建，本文算法代码已完全开源，欢迎社区试用、反馈与共建。
    

未来，我们将继续探索更长 horizon 的任务规划与更复杂的真实场景泛化，期待 π-StepNFT 能为通用机器人策略的落地带来新启发。
