# 机器人基础模型 API：Physical Intelligence 的「物理智能层」愿景

Physical Intelligence 发布博客（非 π0.7），提出**机器人也需要 API**：构建完整「物理智能」技术栈成本极高，希望通用机器人基础模型可被广泛调用，从而催生多场景落地。

- **博客链接**：https://www.pi.website/blog/partner?v=1

---

## 核心观点

- 软件领域可调用 LLM API 快速构建应用，机器人领域缺少现成的「物理智能层」——控制器、数据流水线、模型训练均需自建，许多仍是开放研究问题。
- 通用机器人基础模型（如 π0、π*）可提供现成智能层，降低构建实用机器人系统的成本；一旦可被广泛访问，家庭、医院、仓库、工厂等场景的应用会加速涌现。
- PI 与部署方合作，在真实环境与多样化任务下验证模型表现。

---

## 合作伙伴案例（摘要）

**Weave（家庭/洗衣场景）**  
- 聚焦折叠衣物等长程、可变形物体任务。  
- 使用 PI 模型后：折叠质量提升、单件耗时缩短、远程专家干预次数减少。

**Ultra（工业订单打包）**  
- 电商订单打包，工作流与物品差异大，传统自动化僵化。  
- VLA 模型随数据规模提升性能；与 PI 合作后周期时间、成功率改善，吞吐提升、远程干预负担减轻。  
- 行为层面：指令遵循更强、长尾 case 恢复策略更智能。  
- 模型在真实客户仓库中运行，人机协同干预保障准确率，同时持续产生数据用于下一代迭代。

## PI 团队结语

目标与 LLM 类似：让研究者像调用 LLM API 一样使用机器人基础模型驱动自有应用，通过「物理智能层」的便捷访问降低探索新应用、新形态与新场景的成本；更多合作伙伴案例将陆续分享。
