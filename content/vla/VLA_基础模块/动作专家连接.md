# 动作专家连接

## 一、VLA 典型三段式结构

标准 VLA 通常是：

视觉编码器  →  VLM大脑  →  动作专家 (Policy)
 image         reasoning      control

视觉：ViT / SigLIP / DINOv2

大脑：LLM / VLM

动作专家：MLP / Transformer / Diffusion Policy / Flow Matching

二、VLM 和 动作专家怎么“接”？

目前主流有 5种方式。

方式一：直接线性投影（最早期）
结构
VLM hidden state
     ↓ Linear Projection
 Action token
做法

取 VLM 最后一层 hidden state

过一个 Linear 层

输出：

关节角

末端位姿

gripper 开合

代表

Google DeepMind 的 RT-2

早期 BC-based VLA

优点

简单、端到端

缺点

语义空间 → 连续控制空间是“硬对齐”

控制精度差

长时序不稳定

方式二：Action Token 化（离散动作）

这是现在主流之一。

核心思想

把动作变成“词”。

Pick → <MOVE_X_0.2> <MOVE_Z_-0.1> <GRIP_CLOSE>
流程

动作离散化（VQ-VAE / Kmeans）

变成 action vocabulary

让 VLM 直接预测 action token

代表

RT-2

OpenVLA

PaLM-E

优点

与LLM架构自然融合

可以用 next-token 训练

缺点

离散误差

精细控制能力有限

方式三：VLM做“规划”，动作专家做“控制”

这是目前最稳定的工业方案。

结构
VLM → 生成 skill-level plan
       ↓
动作专家 → 输出 low-level control
举例

VLM输出：

1. Move to cup
2. Align gripper
3. Close gripper
4. Lift

动作专家：

输入：视觉特征 + 当前状态 + 当前 skill

输出：连续控制

代表

Google DeepMind 的 RT-1

分层 VLA 系统

多数工业落地方案

优点

稳定

可替换动作专家

可做强化学习 fine-tune

缺点

两阶段训练

语义误差会放大

方式四：Conditioned Policy（条件控制）

这是 diffusion / flow matching 常用的方式。

核心：

VLM 不直接输出动作，而是输出：

condition embedding

然后：

Diffusion Policy( condition = VLM embedding )
结构
image → VLM → z_cond
                ↓
          Diffusion Policy
                ↓
           action trajectory
代表

Diffusion Policy

LingBot-VLA

优点

控制平滑

长轨迹更稳定

精度高

缺点

训练复杂

推理慢

方式五：共享Transformer（最先进）

直接让动作成为 transformer 的输出头。

[image tokens] + [text tokens] + [robot state tokens]
                 ↓
            Shared Transformer
                 ↓
          action head
特点

不分大脑和动作专家

统一 attention

所有模态在同一 token 空间

代表趋势

大一统 VLA

world-model-based VLA

三、信息传递的“数据形式”到底是什么？

VLM传给动作专家通常有 4 种形态：

1️⃣ CLS embedding

一个向量：

z ∈ R^1024

最简单

2️⃣ Token sequence
[z1, z2, z3, ... zn]

动作专家可以 cross-attention

更强表达能力

3️⃣ Skill token
<PICK>
<POUR>
<PUSH>

更高层抽象

4️⃣ Latent plan

输出一个：

future state trajectory embedding

用于 world model 预测

四、目前主流趋势（2025）

现在明显趋势是：

❌ 不再用

纯线性投影

纯action token

✅ 主流

VLM 做高层语义

Diffusion / Flow Matching 做低层控制

中间传递 embedding + skill token

特别是：

Flow Matching 比 Diffusion 更快

Latent Plan 比 单步预测 更稳定




在 VLA 中，Flow Matching 的输入本质是：

当前噪声动作轨迹 + 条件语义 embedding + 机器人状态
→ 预测一个 velocity field（流场）

一、Flow Matching 在机器人里的角色

它不是直接输出 action，而是学习一个：

vθ(x_t , t | condition)

即：

x_t ：当前“带噪轨迹”

t ：时间

condition ：视觉/语言/状态

输出：把轨迹往真实轨迹方向推的速度
